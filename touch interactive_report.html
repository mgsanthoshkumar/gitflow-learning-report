Core Learning: Command-Line Syntax

A recurring challenge during this exercise was attempting to run multiple commands on a single line without a proper separator. This is a common pitfall for learners. The key takeaway is to **always run separate commands on separate lines** or chain them correctly with `&&` or `;`.

> git add . git commit -m "..." # Incorrect

> git add .
> git commit -m "..." # Correct

The Gitflow Journey

Phase 1: Project Initialization

The repository is created, and the two main branches, `master` and `develop`, are established.

Phase 2: Feature Development

A new feature is developed in an isolated `feature/*` branch and then merged into `develop`.

Phase 3: Major Release (v1.0.0)

A `release/*` branch is created for stabilization, then merged into both `master` (and tagged) and `develop`.

Phase 4: Emergency Hotfix (v1.0.1)

A critical bug in production is fixed on a `hotfix/*` branch, which is created from `master` and merged back into both `master` and `develop`.

Final Result: Repository State

The complete, verifiable history of the repository after all phases are complete.
